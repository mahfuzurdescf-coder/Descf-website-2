# DESCF Website Architecture Plan

## MVP Scope

- Home
- About DESCF
- Mission, Vision & Values
- Leadership & Governance
- Current Work
- Programmes
- Programme Detail Pages
- Strategic Priorities
- Evidence & Resources
- Reports & Publications
- Events / Seminars / Talks
- Partner With Us
- Donate / Support Us
- Contact
- Newsroom / Blog

## CMS Content Types

- Site Settings
- Pages
- Programmes
- Posts
- Authors
- Categories
- Tags
- Resources
- Events
- Team Members
- Media Mentions

## Programme Areas

- Herpetofauna Conservation
- Snake Conservation & Coexistence
- Public Awareness & Conservation Communication
- Wildlife Documentation & Learning
- Walk With Nature by Sound
- Passive Acoustic Monitoring
- Anuran Bioacoustics
- Climate-Responsive Conservation
- Conservation-Linked Green Enterprise

## Implementation Phases

1. Planning and repository setup
2. Next.js foundation
3. Design system
4. Sanity schema setup
5. CMS-driven pages
6. SEO and accessibility pass
7. Vercel deployment
8. Content QA and launch

## Risks

- Overengineering before content is ready
- Too many CMS schemas too early
- Fake or unverified impact claims
- Premature bilingual routing
- Uploading secrets or build folders to GitHub
